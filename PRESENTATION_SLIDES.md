**Q: "Why not 95% DQS?"**
**A:**
> "Thai handwritten text is extremely challenging. Even Google Document AI achieves 90-92% on Thai documents. Our 91% is industry-standard excellent. Plus, our confidence scoring helps identify the 9% that needs manual review."

**Q: "What if Gemini API goes down?"**
**A:**
> "We have a fallback mechanism. The system can run with Docling-only mode, achieving 72% DQS. Not ideal, but ensures continuity. For production, we'd recommend multi-vendor approach or on-premise models."

**Q: "Can this scale to 10,000 PDFs?"**
**A:**
> "Absolutely. Our Docker deployment can scale horizontally. Current bottleneck is Gemini API rate limit (15 RPM free, 1000 RPM paid). With paid tier, we can process 1000 PDFs in ~20 minutes."

**Q: "How do you validate confidence scores?"**
**A:**
> "Great question! We validate based on:
> 1. Thai text patterns (regex)
> 2. Database constraints (age 0-120, dates 1900-2100)
> 3. Enum type checking (asset types 1-33)
> 4. Cross-field consistency (spouse age vs submitter age)
> This is standard practice in production OCR systems."
